TL;DR - Resumen Ejecutivo

Para utilizar OpenCode en VS Code con Ubuntu, presiona Ctrl+Esc para iniciar una sesión en el directorio de tu proyecto. Usa el Modo Plan (tecla Tab) para diseñar estrategias sin modificar archivos, y cambia al Modo Build para implementar cambios automáticos. Referencia archivos específicos con la sintaxis @archivo#L37-42 o usa Alt+Ctrl+K para insertar automáticamente el archivo o selección actual. Formula prompts detallados como si instruyeras a un desarrollador junior, itera en Modo Plan antes de construir, y verifica siempre el modo activo antes de ejecutar comandos para evitar modificaciones accidentales.

1.1 Validación de Componentes Instalados

OpenCode CLI

Verifica la instalación global en Ubuntu

opencode --version
[1]

Extensión SST

Confirma la correcta instalación de la extensión en VS Code

Ctrl+Shift+X → sst-dev.opencode
[2]

Modelo Big Pickle

Selección del modelo gratuito por defecto

opencode auth login
[1]

Verificación Rápida

Ejecuta el comando opencode en una terminal para verificar que la TUI se inicia correctamente. Deberías ver la interfaz de usuario con el prompt de OpenCode.

1.2 Acceso Inicial

1

Lanzamiento desde Terminal Integrada

Utiliza el atajo Ctrl+Esc en VS Code para iniciar OpenCode en el directorio de tu proyecto actual. La extensión SST detectará automáticamente el directorio de trabajo.

# Atajo de teclado
Ctrl + Esc
[2]
2

Activación de la Interfaz TUI Nativa

La interfaz de usuario textual (TUI) se activa automáticamente, mostrando un layout dividido con área de conversación, barra de entrada y paneles de estado.

Características de la TUI:
  • Soporte para temas y colores
  • Integración con LSP para análisis semántico
  • Comandos específicos con prefijo /
  • Atajos de teclado configurables
[1]

2. Modos de Operación Fundamentales

Plan

2.1 Modo Plan (Planificación)

Definición de Estrategias

El Modo Plan permite diseñar estrategias sin modificar archivos. El agente analiza solicitudes complejas y genera planes detallados de implementación.

Principio: "Primero planificar, luego ejecutar"
[1]

Revisión y Refinamiento

Capacidad para analizar críticamente planes, solicitar modificaciones específicas y refinar estrategias iterativamente antes de la implementación.

Ciclo de Planificación

1
Solicitud Inicial
Descripción del problema
2
Generación de Plan
Análisis y propuesta
3
Revisión Crítica
Validación y feedback
4
Refinamiento
Ajustes y mejoras

Alternancia de Modos

Presiona la tecla Tab para alternar instantáneamente entre Modo Plan y Modo Build. Verifica visualmente el indicador de modo antes de ejecutar comandos.

[1]
Build

2.2 Modo Build (Construcción)

Implementación Directa

El agente puede modificar directamente el sistema de archivos para implementar características, corregir errores o refactorizar código.

⚠️ Cambios automáticos en archivos
[1]

Generación Automática

Transforma descripciones de alto nivel en implementaciones completas, incluyendo múltiples archivos y actualizaciones coordinadas.

Aplicación en Tiempo Real

Los cambios se aplican atómicamente al sistema de archivos, con actualización inmediata en el editor de VS Code.

3. Atajos y Navegación en VS Code

3.1 Gestión de Sesiones

Lanzamiento Rápido

Ctrl + Esc

Inicia o enfoca una sesión de OpenCode existente

[2]

Nueva Sesión

Ctrl + Shift + Esc

Crea una sesión independiente en nueva pestaña

Enfoque Sesión

Ctrl + Esc

Trae al frente una sesión existente sin crear nueva

Ventajas de Sesiones Múltiples

  • Contextos aislados
  • Trabajo paralelo
  • Sin contaminación cruzada
  • Diferentes tareas simultáneas
  • Aislamiento de errores
  • Memoria independiente

3.2 Referencias de Archivos

1

Atajo para Inserción (Alt+Ctrl+K)

Inserta automáticamente una referencia al archivo actual o selección con la sintaxis @archivo#Líneas

# Sintaxis generada
@NombreArchivo#L37-42
[2]
2

Búsqueda Difusa con @

Comienza a escribir @ seguido de parte del nombre del archivo para activar el autocompletado inteligente

# Ejemplo de uso
@userctrl → UserController.js
@auth → authenticationMiddleware.ts
3

Contexto Automático

OpenCode recibe automáticamente:

  • Archivo activo en la pestaña del editor
  • Texto seleccionado (resaltado)
  • Posición del cursor en el código
